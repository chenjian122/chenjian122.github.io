Sring Cloud服务注册中心高可用集群

在Sring Cloud Eureka(服务注册与发现) 这篇文章中我们已经介绍了单点的服务注册中心，下面我们来看看该如何构建高可用的Eureka Server集群。

Eureka Server的高可用

Eureka Server除了单点运行之外，还可以通过运行多个实例，并进行互相注册，实现完全对等的方式来实现高可用，任何一个节点挂掉都不会影响服务的注册与发现。所以我们只需要将Eureke Server配置其他可用的serviceUrl就能实现高可用部署。

首先我们需要修改host文件 添加peer1，peer2，peer3的转换

Windows的hosts路径为：C:\Windows\System32\drivers\etc
linux的hosts路径为：/etc/hosts
这样的话host就都转换成本机了

下面我们分别创建application-peer1.properties，application-peer2.properties，application-peer3.properties3个文件分别作为服务中心的配置，用—分割每个配置文件的内容

创建Spring Boot应用分别启动peer1、peer2、peer3三个服务注册中心

然后修改–spring.profiles.active=peer2 ，–spring.profiles.active=peer3 启动

当然你也可以打成jar包启动

然后我们可以访问peer1的注册中心：http://localhost:8761/ ，peer2的注册中心：http://localhost:8762/ ，peer3的注册中心：http://localhost:8763/

客户端使用配置

只需要将之前的单节点的配置改为多节点即可，

启动该服务后，通过访问http://localhost:8761/、http://localhost:8762/ 和 http://localhost:8763/ ，可以观察到eureka-client同时被注册到了peer1、peer2和peer3上。若此时断开peer1，由于eureka-client同时也向peer2、peer3注册，因此在peer2、peer3上其他服务依然能访问到eureka-client，从而实现了高可用的服务注册中心。
